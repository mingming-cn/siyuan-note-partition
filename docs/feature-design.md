# 思源分区插件功能设计

## 1. 目标

在不切换真实思源工作空间的前提下，提供“工作分区”能力。

每个分区：
- 拥有自己关联的顶级笔记本集合
- 仅显示自己关联的顶级笔记本
- 可被快速切换

该插件提供的是界面层面的视图隔离，不是安全隔离。

## 2. 核心功能

### 2.1 分区模式

- 插件启动后存在一个“当前分区”
- 当前分区决定：
  - 哪些顶级笔记本可见
  - 顶部左侧按钮显示什么分区名称
  - 新建顶级笔记本归属到哪个分区
- 切换分区后，应尽快按分区规则打开关联笔记本并关闭未关联笔记本

### 2.2 分区管理

分区管理界面需要支持：
- 添加分区
- 删除分区
- 重命名分区
- 切换分区
- 管理分区与笔记本的关联关系

约束：
- 至少保留一个分区
- 最后一个分区不允许删除
- 一个笔记本可以同时关联多个分区

### 2.3 默认分区

首次安装或首次初始化插件时：
- 自动创建一个“默认分区”
- 默认分区自动关联当前全部顶级笔记本
- 默认分区作为初始激活分区

### 2.4 分区与笔记本关联

当前产品模型固定为“按顶级笔记本关联”：
- 每个分区维护自己的 `notebookIds`
- 当前分区仅显示其关联的顶级笔记本
- 不扩展到路径级过滤
- 不扩展到单篇笔记级关联

### 2.5 新建笔记本自动关联当前分区

当用户在当前分区下新建顶级笔记本时：
- 新笔记本应自动关联到当前分区
- 自动关联只做“追加关联”，不应移除它在其它分区中的归属

## 3. 数据模型设计

### 3.1 分区定义

```ts
interface PartitionConfig {
  id: string;
  name: string;
  notebookIds: string[];
  sort: number;
  createdAt: number;
  updatedAt: number;
}
```

说明：
- `notebookIds` 是唯一过滤条件

### 3.2 插件状态

```ts
interface PluginState {
  version: number;
  activePartitionId: string;
  partitions: PartitionConfig[];
}
```

建议：
- 使用 `version` 支持后续迁移
- 所有持久化数据统一保存在一个插件存储键下

## 4. 交互设计

### 4.1 顶栏入口

当前主入口为顶部左侧按钮：
- 显示内容为“图标 + 当前分区名称”
- 点击后打开菜单
- 菜单支持快速切换分区
- 菜单支持打开分区管理界面

### 4.2 分区管理弹窗

当前管理界面分为左右两栏：

左侧：
- 分区列表
- 新增分区按钮
- 删除分区入口

右侧：
- 分区名称编辑
- 保存按钮
- 笔记本关联栅格列表

删除规则：
- 当仅剩一个分区时禁用删除，并提示原因

## 5. 行为规则

### 5.1 初始化规则

插件加载时：
1. 读取存储
2. 若无有效数据，则读取当前全部顶级笔记本
3. 创建“默认分区”
4. 将全部顶级笔记本 ID 关联到默认分区
5. 设置默认分区为激活分区

### 5.2 分区切换规则

切换分区时：
1. 更新 `activePartitionId`
2. 刷新顶部按钮显示
3. 刷新分区相关 UI
4. 根据当前分区打开关联笔记本
5. 关闭未关联笔记本

当前实现策略：
- 通过 notebook API 打开当前分区关联的笔记本
- 通过 notebook API 关闭当前分区未关联的笔记本
- 不对路径级文档树做额外过滤

### 5.3 删除规则

删除分区时：
- 如果当前总数为 1，则拒绝删除
- 如果删除的是当前激活分区，则自动切换到剩余分区中的第一个

### 5.4 新建笔记本规则

用户新建顶级笔记本时：
- 刷新当前顶级笔记本列表
- 对比新旧笔记本 ID 集合
- 将新增的顶级笔记本加入当前激活分区的 `notebookIds`
- 重新应用当前分区可见性

## 6. 技术实现结构

当前建议保持以下模块分工：

- `src/core/partition-store.ts`
  - 负责状态读取、保存、迁移、默认值初始化

- `src/core/partition-service.ts`
  - 负责分区增删改查
  - 负责笔记本与分区的关联规则
  - 负责“至少保留一个分区”等业务规则

- `src/core/notebook-visibility.ts`
  - 负责根据当前分区打开或关闭顶级笔记本

- `src/partition-panel.svelte`
  - 负责分区管理界面

- `src/index.ts`
  - 负责插件生命周期、顶部按钮、菜单交互、事件监听与新建笔记本同步

## 7. 当前实现状态

已实现：
- 默认分区初始化
- 分区增删改查
- 分区与顶级笔记本关联管理
- 顶部左侧按钮快速切换
- 分区管理弹窗
- 基于顶级笔记本的可见性控制
- 新建顶级笔记本自动关联到当前分区的同步逻辑

未实现或未完善：
- 分区配置导入导出
- 手动打开/关闭笔记本与分区规则冲突时的更强反馈
- 更完善的新建笔记本事件调试与提示

## 8. 风险与注意事项

- “隐藏”不等于权限控制，文案必须明确
- 打开/关闭顶级笔记本会直接影响用户当前界面状态
- 新建笔记本自动关联依赖事件触发和笔记本列表刷新节奏，需要持续关注稳定性

## 9. 当前结论

基于现有需求，当前稳定的产品模型是：

- 分区是用户自定义的工作视图
- 每个分区按“顶级笔记本集合”控制可见性
- 首次安装自动创建“默认分区”，并关联全部顶级笔记本
- 保证至少保留一个分区
- 顶部左侧按钮是唯一主入口
- 新建顶级笔记本应自动归属当前分区
